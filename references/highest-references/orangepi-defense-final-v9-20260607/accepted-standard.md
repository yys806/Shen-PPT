This reference pack records the accepted Shen-PPT visual and execution standard from 2026-06-07. The active baseline is the v9 productized deck approved after evidence density, icon placement, no-animation, slide-card, speaker-note, and productized delivery refinements.

Use this pack as the highest-quality reference and acceptance benchmark only. It is **not a reusable template**. Do not copy project-specific OrangePi content, metrics, screenshots, names, conclusions, slide order, hardware narrative, or page compositions into unrelated decks.

For machine-style production parameters, use `../../parameter-spec.md` together with this visual reference. The parameter spec is binding for default generation unless the user explicitly approves a different system.

## Highest-Reference Standard

- Build editable PPT elements directly through Shen-PPT's approved PPT construction path
- Keep the default dark green/black academic engineering style with amber accent, cyan numbers, muted subtitles, and complete real evidence images unless the user chooses another style
- Use productized slide cards before construction: purpose, claim, source evidence, layout family, visual asset, speaker note, and QA risk
- Use dense vertical rhythm: top rule around `y=100`, body starts around `y=112`, content normally reaches `y=640-660`, bottom rule at `y=670`, footer/page number below it
- Treat `y=660` as the normal content-bottom target and avoid large empty lower bands
- Keep top-right module buttons fixed, two-line, and consistent: number on the first line, four-character label on the second line
- Use formal, material-grounded copy without Chinese full stops on slides
- Prefer real screenshots, result tables, charts, wiring images, terminal evidence, and report figures; use AI only as local illustration assets when useful
- Show real/source images complete with contain placement; never crop real evidence accidentally
- Use one semantic line-icon system only when icons clarify structure: section divider identity, evidence labels, process rows, metric labels, and conclusion cues
- Keep directory pages icon-free by default
- For same-role batches, either every peer item has the same icon treatment or the whole batch omits icons
- Section divider pages use large aligned section number/title and concise talking points, not forced card grids
- Body pages have one section title and one formal subtitle only
- Process pages need visible arrows with enough spacing; horizontal rows place number/title on the left and explanation on the right
- Boxes, image cards, and proof objects should feel intentionally filled; enlarge evidence and add source-grounded detail before adding decoration
- Final thank-you page is centered, no-card, and no-frame, with presenter line below the thanks text
- Standard Shen-PPT decks have no animations, no object build effects, and no slide transitions
- Every final deck must include the editable PPTX plus two compact Markdown files: speaker script and likely Q&A

## Final Package Standard

The accepted delivery form is one PPTX plus two Markdown documents placed next to it:

- `{deck-title}.pptx`
- `{deck-title}_讲稿.md`
- `{deck-title}_问答.md`

The Markdown documents must be generated from the final PPT and original source materials. They should be compact, slide-aligned or question-aligned, and free of extra blank lines, boilerplate prefaces, generated-document explanations, and unsupported claims.

## Style Sample Implication

Style sample decks are separate from this highest reference. Samples provide reusable skins and parameter examples; this pack provides the quality bar. Other visual styles may change palette, background treatment, accent color, logo policy, representative images, and project-specific structure, but they must meet the accepted density, navigation discipline, icon discipline, editable-object rule, no-animation rule, and final package rule.
